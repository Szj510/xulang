# GIF 分享、真实画框与 3D 模板设计

## 目标

在已合并的叙事长廊版本上推进三项下一轮能力：

- 用体积更可控的动态 GIF 分享替代“图片内嵌 HTML”作为默认轻量分享方式，保留 HTML 作为备用导出。
- 让新增画框不再只是纯色边框，而是有可感知的木纹、金属划痕、旧纸污渍和胶片孔细节。
- 增加一个“立体展墙”章节模板，用 Flutter 原生 `Transform` 与深度参数呈现 2.5D 纵深，不引入真 3D 引擎。

## 范围

- GIF v1 生成 540px 宽、最多 12 帧、每帧约 700ms 的故事预览；优先使用缩略图，缺失时跳过损坏图片。
- GIF 文件通过系统分享面板分享，HTML 导出仍保留在菜单中。
- 画框纹理全部由 `CustomPainter` 程序化绘制，避免新增大图片素材。
- 3D 模板作为新的 `GalleryLayout.depthWall`，编辑器可选择，播放器/预览共用 `LayoutResolver` 结果。

## 非目标

- 不做视频导出。
- 不做在线模板市场。
- 不做真 3D 自由漫游。
- 不保证 GIF 完全复刻播放器所有透视细节；GIF 目标是小体积、可分享的动态预览。

## 测试

- GIF 导出：验证生成 `.gif`、文件非空、帧数据包含 GIF 头，且损坏/缺失图片不会崩溃。
- 3D 模板：验证节点横竖屏都有顺序、深度递增/递减合理、内容范围可导航。
- 画框：现有全枚举 widget 测试继续覆盖新增纹理画框；补充纹理 painter key 断言。
