# 叙廊：连续叙事长廊 V2 设计规格

## 设计基准

- Figma：[叙廊 · 连续叙事长廊 V2](https://www.figma.com/design/mL11DIjbdRs23TazvOAyi2)
- 视觉基准：初版“山海之间”效果图的左侧章节轨道、主图透视、前中后景层叠和黑色空间。
- 主选版式参考：[`exec-2432a883-91a6-4673-8c8d-4814b2cb13ed.png`](C:/Users/suzij/.codex/generated_images/019eeae9-ca73-7c52-a8b7-0a42de829178/exec-2432a883-91a6-4673-8c8d-4814b2cb13ed.png)。保留现有四种版式，并将此构图新增为第五种“故事路径”。
- 交互基准：用户横向拖动的是同一条连续相机轨道，不是分页容器，也不是点击按钮后切换图片。

## 已确认的方向

采用连续镜头轨道方案：每章是一条标准化 `0…1` 的空间相机路径。照片作为路径上的场景节点，在进入、成为焦点和退出时连续插值位置、尺寸、Y 轴透视、旋转、透明度和景深。手指位移直接写入相机进度，松手后按末速度惯性减速；不自动分页，不强制吸附。

显式上一项/下一项按钮保留为无障碍备用入口，不再作为主要观看方式。

## 当前实现的问题与根因

1. 横屏编辑器使用固定高度检查器，内部内容超过可用空间，但外层没有完整滚动区域，导致底部按钮不可达。
2. `GallerySize` 虽保存到数据层，但部分模板在 `LayoutResolver` 中使用固定尺寸，覆盖了小/中/大选择。
3. 画框仅在图片内部绘制窄边框，并被裁切或被图片覆盖，视觉差异不明显；邮票边也没有独立轮廓。
4. 播放器使用横向手势修改 `focusIndex`，本质仍是离散换图；`SceneCanvas` 只在换图时重新排列节点，没有连续相机状态。
5. 当前主图透视较弱，缺少初版效果图中的前景退出、中景成为主体、远景进入的连续空间关系。

## 播放器交互

### 手势状态机

- 手势开始时记录缩放值与初始坐标。
- 累积位移超过 8dp 后锁定主方向。
- `scale > 1.01` 时锁定为图片平移；禁止章节和轨道导航。
- 1× 状态下：横向位移驱动 `cameraProgress`，纵向位移交给章节切换。
- 方向一旦锁定，直到所有手指抬起前不改变。
- 横向拖动比例：`progressDelta = -dx / viewportWidth / dragSpan`，`dragSpan` 由照片数量和模板密度计算。
- 松手时根据像素速度启动摩擦模拟；到达 `0` 或 `1` 时产生克制的边界回弹。

### 连续场景节点

每个展示位置生成一个 `NarrativeKeyframe`：

- `focusProgress`：该图片成为叙事主体的轨道位置。
- `enterTransform`：从远景或侧后方进入。
- `focusTransform`：主体位置、尺寸和裁切焦点。
- `exitTransform`：进入前景后从另一侧离开。
- `depth`：决定透视、阴影、模糊级别和绘制顺序。

同一进度至少同时可见 2–3 个相邻节点，保证上一幅退出、当前幅聚焦、下一幅进入的连续性。所有章节模板只改变轨道编排语言，不改变连续相机模型。

### 第五种模板：故事路径

“故事路径”严格继承主选参考图的视觉语言：

- 4–8 张照片沿一条纵向弯曲的叙事路径交错分布，而不是规则网格。
- 主图使用相纸或邮票边，次图可以轻度旋转、模糊和压暗，形成前中后景。
- 路径节点显示序号和短标题，例如“01 启程”“02 巷遇”；文字必须与对应照片的焦点关键帧绑定。
- 路径绘制在照片后方，使用低对比细线和实心节点，不能穿过主体照片的可读区域。
- 初始状态展示章节的整体构图；用户横向拖动后，相机沿这条曲线依次推进、放大和聚焦各节点。
- 拖动过程中仍同时保留相邻照片，形成“整张故事地图正在被镜头探索”的效果，而非单图轮播。
- 提供“回到全景”控制；它只改变相机进度和缩放，不改变图片顺序。

现有 `hero / filmstrip / diptych / collage` 四种模板继续保留；领域枚举新增 `storyPath`，数据库以稳定字符串保存模板值，避免后续新增枚举导致旧数据错位。

### 方向与章节

- 上下滑动仍切换章节。
- 横向轨道到达章首或章尾后继续拖动不会误切章节。
- 横竖屏切换重新解析节点几何，但保留章节 ID、`cameraProgress`、缩放和焦点。
- 单击隐藏/显示控件；章节标题、注释和导航控件使用渐变或半透明底板保障对比度。

## 编辑器

### 横屏结构

- 顶部：应用栏与章节栏。
- 主区域：左侧连续轨道预览，右侧属性面板。
- 属性面板使用独立纵向滚动；所有控件必须能滚动到可见区域。
- 窄横屏下，属性面板最小宽度 280dp；不足时允许整个主区域垂直滚动，不遮挡系统导航栏。

### 实时样式反馈

- 小/中/大分别映射为模板基准面积的约 `0.58 / 0.78 / 1.0`，影响节点焦点宽高和轨道间距。
- 无框：仅保留极细防丢失轮廓。
- 细线：1dp 高对比描边，绘制在图片外侧。
- 相纸：明显留白，底部留白大于其他三边。
- 邮票边：使用独立 `CustomPainter` 绘制规则齿孔轮廓，不能用普通实线边框代替。
- 每次选择立即更新预览；保存继续走现有自动保存和撤销栈。

## 架构调整

- 新增 `NarrativeTrackResolver`：将章节、视口和模板解析为带关键帧的 `ResolvedNarrativeTrack`。
- 新增 `NarrativeCameraController`：持有 `progress`、速度、缩放优先级和方向锁定状态。
- `SceneCanvas` 改为纯渲染器：接收轨道与进度，不持有导航业务状态。
- 编辑器和播放器共用同一 `NarrativeTrackResolver`，避免预览与播放结果不同。
- 保留原 `LayoutResolver` 作为静态节点几何工具，逐步由轨道解析器组合，而不是直接删除。

## 测试与验收

- 单元测试：关键帧确定性、尺寸映射、轨道边界、摩擦衰减、方向锁定、缩放优先级。
- Widget 测试：横屏属性面板可滚动到底；四种画框产生不同绘制结构；小/中/大得到不同节点面积。
- Golden：五种模板在轨道 `0%、35%、70%` 的横竖屏画面，覆盖墨色/纸张和减少动态效果；“故事路径”额外覆盖全景与节点聚焦状态。
- Android 集成：连续横拖能在一章内自然经过至少三张图片；按钮不遮挡；旋转后保持相机进度。
- 性能：相机进度变化不触发数据库写入；预加载当前焦点前后各两张；1080p 中端机目标 60fps。

## 明确不做

- 不引入真 3D 或游戏引擎。
- 不提供自由漫游和任意方向无限画布。
- 不使用分页吸附作为主要交互。
- 不改变现有离线存储、导入、章节数据和 v1 边界。
